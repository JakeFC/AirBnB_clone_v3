# AirBnB Clone - Restful API
The console is the first segment of the AirBnB project at Holberton School that will collectively cover fundamental concepts of higher level programming. The goal of AirBnB project is to eventually deploy our server a simple copy of the AirBnB Website(HBnB). A command interpreter is created in this segment to manage objects for the AirBnB(HBnB) website.

#### Functionalities of this command interpreter:
* Create a new object (ex: a new User or a new Place)
* Retrieve an object from a file, a database etc...
* Do operations on objects (count, compute stats, etc...)
* Update attributes of an object
* Destroy an object

## Environment
This project is interpreted/tested on Ubuntu 14.04 LTS using python3 (version 3.4.3)

## Installation
* Clone this repository: `git clone "https://github.com/szbrooks2017/AirBnB_clone_v3"`
* Access AirBnb directory: `cd AirBnB_clone_v3`
* Run hbnb(interactively): `./console` and enter command
* Run hbnb(non-interactively): `echo "<command>" | ./console.py`

## File Descriptions
[console.py](console.py) - the console contains the entry point of the command interpreter. 
List of commands this console current supports:
* `EOF` - exits console 
* `quit` - exits console
* `<emptyline>` - overwrites default emptyline method and does nothing
* `create` - Creates a new instance of`BaseModel`, saves it (to the JSON file) and prints the id
* `destroy` - Deletes an instance based on the class name and id (save the change into the JSON file). 
* `show` - Prints the string representation of an instance based on the class name and id.
* `all` - Prints all string representation of all instances based or not on the class name. 
* `update` - Updates an instance based on the class name and id by adding or updating attribute (save the change into the JSON file). 

[app.py](/api/v1/app.py) - creates the Flask instance which makes the data accessible through browser commands
* `def teardown(error)` - closes the storage instance between requests
* `def error_handler(error)` - returns a 404 page on request error

#### `models/` directory contains classes used for this project:
[base_model.py](/models/base_model.py) - The BaseModel class from which future classes will be derived
* `def __init__(self, *args, **kwargs)` - Initialization of the base model
* `def __str__(self)` - String representation of the BaseModel class
* `def save(self)` - Updates the attribute `updated_at` with the current datetime
* `def to_dict(self)` - returns a dictionary containing all keys/values of the instance

Classes inherited from Base Model:
* [amenity.py](/models/amenity.py)
* [city.py](/models/city.py)
* [place.py](/models/place.py)
* [review.py](/models/review.py)
* [state.py](/models/state.py)
* [user.py](/models/user.py)

#### `/models/engine` directory contains File Storage class that handles JSON serialization and deserialization :
[file_storage.py](/models/engine/file_storage.py) - serializes instances to a JSON file & deserializes back to instances
* `def all(self)` - returns the dictionary __objects
* `def new(self, obj)` - sets in __objects the obj with key <obj class name>.id
* `def save(self)` - serializes __objects to the JSON file (path: __file_path)
* `def reload(self)` -  deserializes the JSON file to __objects

#### `/api/v1/views` directory contains all routes and their functions for the API functionality:
[__init.py__](/api/v1/views/__init__.py) - defines a blueprint for the Flask object that prefixes all routes with '/api/v1'

[index.py](/api/v1/views/index.py) - Contains the /status and /stats routes for overall info on the database
* `GET /status` - returns 'status': 'OK' if working correctly
* `GET /stats` - returns a dictionary of object counts by type

[amenities.py](/api/v1/views/amenities.py) - Contains all routes and functions for accessing amenities
* `GET /amenities` - retuns all amenity objects as a list of dictionaries
* `GET /amenities/<amenity_id>` - returns a dictionary of the object with given id
* `DELETE /amenities/<amenity_id>` - removes the object with matching id
* `POST /amenities` - creates a new amenity from given JSON dictionary
* `PUT /amenities/<amenity_id>` - updates the object of given id with the given JSON dictionary values

[cities.py](/api/v1/views/cities.py) - Contains all routes and functions for accessing cities
* `GET /states/<state_id>/cities` - retuns all city objects linked to state with given id as a list of dictionaries
* `GET /cities/<city_id>` - returns a dictionary of the object with given id
* `DELETE /cities/<city_id>` - removes the object with matching id
* `POST /states/<state_id>/cities` - creates a new city from given JSON dictionary linked to state with given id
* `PUT /cities/<city_id>` - updates the object of given id with the given JSON dictionary values

[places.py](/api/v1/views/places.py) - Contains all routes and functions for accessing places
* `GET /cities/<city_id>/places` - retuns all place objects linked to city with given id as a list of dictionaries
* `GET /places/<place_id>` - returns a dictionary of the object with given id
* `DELETE /places/<place_id>` - removes the object with matching id
* `POST /cities/<city_id>/places` - creates a new place from given JSON dictionary linked to city with given id
* `PUT /places/<place_id>` - updates the object of given id with the given JSON dictionary values

[places_reviews.py](/api/v1/views/places_reviews.py) - Contains all routes and functions for accessing reviews

* `GET /reviews/<review_id>` - returns a dictionary of the object with given id
* `DELETE /reviews/<review_id>` - removes the object with matching id
* `POST /places/<place_id>/reviews` - creates a new review from given JSON dictionary linked to place with given id

[states.py](/api/v1/views/states.py) - Contains all routes and functions for accessing states
